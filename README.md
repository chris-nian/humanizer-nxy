## 项目简介

humanizer-nxy 是一个用于去除文本中 AI 生成痕迹的工具，帮助你将 AI 生成的内容改写得更自然。

**这个工具做的不仅是换词。** 它的核心逻辑是：先判断文章本来应该是什么类型，然后让它真正成为那种文章——去掉所有营销腔、夸大叙事和表演性结构，留下实质内容。

适用于：
- 技术文章、研究解读、产品技术说明的去 AI 化处理
- 营销文案的诚实化改写
- 学习识别 AI 写作的常见模式

## 核心处理逻辑

### 第一步：判断文章应有的调性

这是新版本最重要的改变。AI 生成的内容经常混淆文章类型：一篇技术解读被写成了营销软文，一篇产品介绍被写成了行业报告。单纯修改用词解决不了这个问题。

**技术类/分析类文章**（技术解读、研究介绍、产品技术说明）：

- 调性：克制、精确、有观点但不煽情
- 标题直接描述文章内容，不用宏大叙事
- 结论是基于论据的推断，不是公司口号
- 引用来源明确，不用"行业专家认为"这类模糊归因
- 具体细节（数字、实验数据、英文原文引用）比形容词堆砌更有说服力
- 公司自吹段落（"我们相信…""正在帮助…走完这条路"）一律删除或改为中性陈述

**营销类/推广类文章**（产品介绍页、案例分享）：

- 调性：明确、诚实、直接
- 用具体功能和数据说话，不用形容词堆砌
- 营销腔不一定全部删除，但替换夸张用词

### 第二步：识别结构性问题并重构

结构性问题不能靠改词解决：

| 问题类型 | 识别方式 | 处理方法 |
|---------|---------|---------|
| 标题夸大叙事 | 含"重新定义""赋能""遇见""未来"等 | 改为直接描述文章内容 |
| 公司自吹段落 | "我们相信/致力于/承诺"开头，末尾金句结束 | 删除或改为中性的有判断陈述 |
| 勾选框列表（✅ ❌） | 营销文的格式标志 | 改为普通文字叙述 |
| 末尾金句 | 听起来可以当广告语 | 重写为有前提条件的判断句 |

### 第三步：处理语言层面的 AI 痕迹

| 类别 | 具体问题 | 处理方法 |
|------|---------|---------|
| AI 高频词 | 此外、至关重要、增强、格局、赋能、落地、颠覆 | 删除或替换 |
| 套话结构 | "正是…的体现""不仅仅是…更是…""这不是终点而是新起点" | 删除 |
| 破折号滥用 | 模仿"有力"销售文案 | 改为正常句式 |
| 粗体滥用 | 制造"这里很重要"的感觉 | 只保留术语和对比数据 |
| 三段式 | 强行凑成三项显得"全面" | 改为两项或直接叙述 |
| 抽象主张 | 形容词堆砌代替具体事实 | 删除主张，保留/补充具体事实 |
| 表情符号 | ✅ 🚀 💡 等 | 技术文章中删除 |

## 使用场景示例

### 场景 1：技术文章去 AI 化

**输入（AI 营销软文风格）：**

> 当 Harness Design 遇见 CoStrict：重新定义企业级 AI 编程的可靠性边界
>
> 22 倍的成本差距，换来的却是"不可用"到"生产级"的质变。这背后的关键，正是 Harness Design 理念——一个决定 AI 能否真正落地企业级场景的核心方法论。
>
> 我们相信，未来的软件开发不是"AI 替代人类"，而是"Harness 增强人类"。而 CoStrict，正在帮中国企业走完这条路。

**输出（技术分析风格）：**

> Harness Design 与 CoStrict：长时 AI 编程任务的企业级实践
>
> 完整 Harness 产出的应用包含了单智能体版本没有的 AI 辅助精灵生成器和关卡设计师功能——尽管成本从 $9 上升到了 $200。背后的关键在于 Anthropic 提出的 Harness Design 理念，这个在国内 AI 编程社区讨论较少，却对 AI 能否真正落地企业场景至关重要。
>
> 关键不是追求全自动，而是找到人机协作的最优分工。Harness 的价值在于让开发者能够驾驭更复杂的项目，同时保持可预测的质量和成本。

**修改了什么：**

- 标题去掉"遇见""重新定义"，改为直接描述
- 删除"22 倍成本差距……质变"的夸大表述，改为具体功能对比
- 删除"我们相信……正在帮中国企业走完这条路"的公司自吹金句
- 结尾改为有前提条件的判断句

## 快速检查清单

改写完成前逐项确认：

- [ ] **标题**：有没有宏大叙事词汇（"重新定义""遇见""赋能"）？→ 改为直接描述内容
- [ ] **调性**：原文应是技术文还是营销文？改写后是否一致？
- [ ] **公司段落**：有没有"我们相信/致力于/正在帮助"式表达？→ 删除或改为中性陈述
- [ ] **结语**：最后一句是金句还是判断？→ 金句一律重写
- [ ] **勾选框列表（✅ ❌）** → 改为普通文字叙述
- [ ] **粗体**：是强调术语还是制造"这里很重要"的感觉？→ 后者全部去掉
- [ ] **破折号** → 改为正常句式
- [ ] **三段式结构** → 改为两项或直接叙述
- [ ] **AI 高频词**：此外、至关重要、增强、格局、赋能… → 全部替换或删除
- [ ] **抽象主张**：有没有形容词堆砌代替具体事实？→ 删除主张，保留事实

## 质量评分标准

| 维度 | 评估标准 |
|------|---------|
| **调性准确** | 文章调性与其应有类型一致？技术文不应有营销腔，营销文不应有学术腔 |
| **直接性** | 直接陈述事实，还是绕圈宣告？ |
| **信息密度** | 用具体细节支撑主张，而非形容词堆砌 |
| **真实性** | 听起来像真人说话？无金句、无煽情 |
| **精炼度** | 还有可删减的内容吗？ |

## 常见 AI 词汇清单

**中文高频词：** 此外、至关重要、深入探讨、强调（动词）、增强、培养、格局（抽象用法）、展示、证明（作动词）、宝贵的、充满活力的、赋能、落地、颠覆

**结构性套话：** "正是…的体现""不仅仅是…更是…""从…到…""这不是终点而是新起点""彰显了…的重要意义"

**标题危险词：** 遇见、赋能、重塑、定义、颠覆、边界、格局

## 与旧版本的差异

| 旧版本 | 新版本 |
|-------|-------|
| 主要关注词汇替换 | 先判断调性，再处理结构，最后改词汇 |
| 鼓励加入情绪和"个性" | 技术文章的个性来自精准判断，不来自情绪 |
| 模式清单驱动 | 分层处理：结构性问题 → 内容问题 → 语言问题 |
| 没有区分文章类型 | 技术文和营销文处理逻辑不同 |
| 结语可以保留金句 | 金句一律重写为有前提条件的判断句 |
